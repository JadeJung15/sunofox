# 오너 스튜디오 로그인 이동 설계

## 목표

사이트의 `오너 스튜디오` 링크로 진입한 사용자가 등록된 오너 이메일 계정으로 로그인하면 내부 정본 경로인 `/mv-studio`로 이동하도록 보장합니다.

## 범위

- 기존 이메일, 비밀번호, 입장 코드 로그인 UI를 유지합니다.
- Google/Kakao 로그인 버튼도 현재 Studio 목적지를 OAuth 시작 요청에 전달합니다.
- OAuth 목적지는 서명된 상태 쿠키에 저장하고 콜백 검증 후 복원합니다.
- `next=/mv-studio`가 지정된 로그인 요청은 오너 계정에만 허용합니다.
- 오너 로그인 성공 시 `/mv-studio`를 응답하고 브라우저가 해당 경로로 이동합니다.
- 일반 회원이 오너 스튜디오 진입을 시도하면 로그인 화면에 권한 안내를 표시하고 스튜디오로 이동시키지 않습니다.
- 회원가입, 관리자 화면, MV Studio 내부 기능은 변경하지 않습니다.

## 구조와 데이터 흐름

1. 내비게이션의 `오너 스튜디오` 링크는 `/login?next=/mv-studio`를 유지합니다.
2. 로그인 화면은 현재 URL의 `next` 값을 로그인 API로 전달합니다.
3. 로그인 API는 계정 인증과 승인 상태를 확인한 뒤 역할을 판별합니다.
4. 요청 목적지가 MV Studio이고 계정 역할이 `owner`가 아니면 권한 오류를 반환합니다.
5. 오너 계정이면 세션 쿠키와 함께 `/mv-studio`를 반환합니다.
6. 브라우저는 API 응답의 안전한 내부 경로로 이동합니다.

소셜 로그인에서는 같은 목적지를 OAuth state 쿠키에 포함하고, 공급자 콜백에서 쿠키 서명과 만료를 검증한 뒤 이동 경로를 복원합니다.

## 오류 처리

- 일반 회원: 제작자 전용 계정이 필요하다는 한국어 안내를 로그인 폼에 표시합니다.
- 잘못된 비밀번호, 미승인 계정, 잘못된 입장 코드는 기존 오류 처리를 유지합니다.
- 외부 URL 또는 비정상 `next` 값은 기존 내부 경로 정규화 로직으로 `/` 처리합니다.
- OAuth state가 없거나 위조·만료된 경우 기존 `state-error` 로그인 안내로 복귀합니다.

## 테스트

- 로그인 API 계약 검사에 오너 계정의 `/mv-studio` 이동 조건을 추가합니다.
- 일반 회원의 `/mv-studio` 요청이 거부되는 조건을 추가합니다.
- Google OAuth 시작부터 콜백까지 `/mv-studio` 목적지가 보존되는 조건을 추가합니다.
- 로그인 화면이 `next=/mv-studio`를 API에 전달하고 성공 응답 경로로 이동하는 기존 연결을 확인합니다.
- 전체 빌드와 저장소 검사, 변경 파일 diff 검사를 수행합니다.
- push 후 Cloudflare Pages 자동 배포와 운영 URL의 공개 로그인 화면을 확인합니다. 실제 인증정보 입력은 자동화하지 않습니다.

## 배포

검증 완료 후 `main`에 일반 커밋하고 `origin/main`으로 push합니다. GitHub 연동 Cloudflare Pages 자동 배포만 사용하며 수동 Pages 배포, DNS 변경, Cloudflare Access 변경은 하지 않습니다.
