# SF Studio Creator Console 개편 설계

- 작성일: 2026-07-13
- 대상: `https://sunofox.com/mv-studio`
- 범위: SF 홈, Workflow MD 가져오기, 콘티 제작, 사용방법
- 방향: 작업 효율 70% + Anime OST 브랜드 감성 30%
- 승인안: Creator Console

## 1. 목표

기존 SF Studio의 기능과 저장 구조를 유지하면서, 화면을 현대적인 제작 도구 형태로 재구성한다.

핵심 목표는 다음과 같다.

1. 현재 작업과 다음 행동을 한눈에 확인할 수 있게 한다.
2. 홈, 가져오기, 콘티, 사용방법의 탐색 구조와 상태 표현을 통일한다.
3. 버튼과 카드의 시각적 우선순위를 명확히 한다.
4. 실제 후보 이미지가 Anime OST 브랜드 감성을 담당하게 한다.
5. 데스크톱과 모바일에서 각각 적합한 작업 구조를 제공한다.

## 2. 현재 화면 진단

현재 화면은 기능이 충분하지만 다음 문제로 인해 오래된 관리 도구처럼 보인다.

- 흰색 카드, 테두리, 작은 버튼이 비슷한 무게로 반복된다.
- 저장, 외부 도구, 백업, 탭 버튼이 한 줄에 나열되어 핵심 행동이 묻힌다.
- 컷 목록, 프롬프트, 이미지, 속성 정보의 시선 우선순위가 약하다.
- 메뉴별 화면 구조가 달라 사용자가 매번 위치를 다시 파악해야 한다.
- 저장 완료, 미저장, 로딩, 오류 상태가 작업 흐름 안에서 충분히 강조되지 않는다.
- 데스크톱 구조를 모바일 폭에 축소하는 방식으로는 실제 조작성이 떨어진다.

## 3. 설계 원칙

### 3.1 한 화면에 하나의 핵심 행동

각 화면에서 가장 중요한 행동 하나만 Primary 버튼으로 표시한다.

- SF 홈: `마지막 작업 이어하기`
- Workflow 가져오기: `검증 후 작업 보드 열기`
- 콘티 제작: `보내고 다음`
- 사용방법: 현재 문제에 맞는 해결 절차 열기

저장, 백업, 외부 도구, 초기화 등은 Secondary 또는 Ghost 단계로 낮춘다.

### 3.2 인터페이스는 조용하게, 작업 이미지는 선명하게

별도 장식 일러스트나 게임형 효과를 추가하지 않는다. 실제 후보 이미지와 프로젝트 커버가 브랜드 분위기를 만든다.

### 3.3 상태는 색상과 문구를 함께 사용

색상만으로 저장, 경고, 오류를 구분하지 않는다. 아이콘 또는 상태 문구를 함께 제공한다.

### 3.4 기존 기능과 데이터 계약 보존

현재 DOM ID, 저장 포맷, 프로젝트 복원, 이미지 저장, Bridge 연결을 최대한 유지한다. UI 개편을 데이터 마이그레이션 작업으로 확대하지 않는다.

## 4. 공통 정보 구조

### 4.1 데스크톱

Creator Console은 다음 네 영역으로 구성한다.

| 영역 | 역할 |
|---|---|
| 좌측 글로벌 레일 | SF 홈, 콘티 제작, Workflow 가져오기, 사용방법, 설정 이동 |
| 상단 명령 바 | 현재 프로젝트, 마지막 저장, 오류 수, 외부 도구, 핵심 저장 행동 |
| 중앙 작업 캔버스 | 현재 화면의 주 작업, 이미지, 프롬프트, 검증 결과 |
| 우측 컨텍스트 패널 | 현재 컷 또는 현재 입력과 직접 관련된 속성 및 다음 행동 |

콘티 제작 화면에서만 글로벌 레일과 중앙 캔버스 사이에 컷 목록을 표시한다.

### 4.2 모바일

데스크톱의 네 영역을 좁게 축소하지 않고 다음과 같이 재배치한다.

- 글로벌 레일 → 하단 고정 메뉴
- 컷 목록 → 검색 가능한 드로어
- 중앙 캔버스 → 전체 폭 작업 영역
- 우측 컨텍스트 패널 → 바텀시트
- 저장 상태 → 상단 고정 상태 영역

터치 대상은 최소 44px 높이를 확보한다.

## 5. 메뉴별 설계

### 5.1 SF 홈

홈은 소개 화면이 아니라 작업 결정 화면으로 사용한다.

표시 우선순위:

1. 마지막 작업 이어하기
2. 최근 프로젝트
3. Workflow MD로 새 작업 시작
4. 이미지로 새 작업 시작

기존 브랜드 설명은 한두 문장으로 줄인다.

### 5.2 Workflow MD 가져오기

입력과 결과를 분리하지 않고 한 화면에서 함께 확인한다.

- 왼쪽 또는 중앙: Markdown 원문 입력
- 중앙 미리보기: 감지된 프로젝트명, 오디오 길이, 컷 수
- 오른쪽 검증 패널: 정상, 경고, 오류 컷 목록
- 보조 옵션: profile, sref, model 설정을 접힌 패널로 제공

오류가 있어도 정상 컷은 유지하며, 오류가 발생한 컷으로 바로 이동할 수 있어야 한다.

### 5.3 콘티 제작

주 반복 흐름을 `컷 선택 → 이미지/프롬프트 확인 → 전송 → 다음 컷`으로 고정한다.

화면 구성:

- 좌측 컷 목록: 썸네일, 컷 번호, 짧은 가사, 저장 상태
- 중앙 상단: 타임라인 길이, 평균 컷, 저장 수, 오류 수
- 중앙 본문: 현재 후보 이미지와 프롬프트
- 우측 패널: 타임코드, 길이, MJ/Grok 준비 상태, Bridge 상태
- 핵심 행동: `보내고 다음`

백업, CSV, 전체 프롬프트 복사 같은 저빈도 기능은 더보기 메뉴 또는 보조 도구 영역으로 이동한다.

### 5.4 사용방법

긴 기능 설명 대신 실제 문제 해결 중심으로 구성한다.

기본 항목:

- Workflow가 인식되지 않을 때
- Bridge가 준비되지 않을 때
- 이미지가 저장되지 않을 때
- 외부 도구 전송이 실패할 때
- 백업을 복원할 때

현재 화면에서 도움말을 열면 해당 화면의 항목을 먼저 보여준다.

## 6. 시각 시스템

### 6.1 색상

| 용도 | 색상 |
|---|---|
| Canvas | `#080B11` |
| Panel | `#111722` |
| Raised surface | `#171F2D` |
| Primary action | `#8B78F6` |
| Bridge/연결 | `#5DD8E4` |
| Saved/정상 | `#68DCA4` |
| Warning | `#F0B85B` |
| Error | `#F07178` |

색상은 기존 CSS 토큰과 커스텀 클래스 체계 안에서 SF Studio 전용 토큰으로 정의한다.

### 6.2 타이포그래피

- 제목: 높은 굵기와 좁은 자간으로 작업 맥락을 빠르게 구분한다.
- 본문: 중간 이하 굵기와 충분한 줄간격을 사용한다.
- 프롬프트와 타임코드: 고정폭 글꼴을 사용한다.
- 작은 상태 문구: 11px 미만으로 축소하지 않는다.

### 6.3 컴포넌트 위계

- Primary: 화면당 하나
- Secondary: 저장, 불러오기, 외부 도구
- Ghost: 탭, 보조 탐색, 접기/펼치기
- Destructive: 초기화와 삭제에만 사용

카드의 테두리와 그림자는 낮은 대비로 유지하고, 선택 상태에서만 청록색 포커스를 제공한다.

### 6.4 모션

150~200ms 범위의 짧은 전환만 사용한다.

- 패널 열림과 닫힘
- 컷 선택 이동
- 저장 완료 토스트
- 상태 변경

`prefers-reduced-motion`에서는 비필수 전환을 제거한다.

## 7. 데이터 흐름과 호환성

현재 저장 구조를 유지한다.

- 프로젝트 텍스트와 상태: `localStorage`
- 후보 이미지: `IndexedDB`
- 현재 세션 접근 상태: `sessionStorage`
- 휴대용 백업: 기존 JSON 내보내기/가져오기

개편 후에도 기존 저장 프로젝트를 그대로 불러올 수 있어야 한다. 저장 포맷 변경이 필요한 요구는 이번 범위에 포함하지 않는다.

Bridge 기준 버전은 `SF 미디어 브릿지 1.5.23`으로 유지한다. 이번 개편은 Bridge 기능이나 권한을 변경하지 않으므로 manifest, ZIP, Studio 요구 버전은 올리지 않는다.

## 8. 상태와 오류 처리

### 8.1 빈 화면

`마지막 작업 이어하기`, `Workflow 가져오기`, `이미지로 시작`을 명확히 분리한다.

### 8.2 로딩

전체 화면 스피너 대신 영향을 받는 패널에 스켈레톤과 진행 문구를 표시한다.

### 8.3 저장

- 저장 중: 상단 상태에 진행 문구 표시
- 저장 완료: 상단 상태 갱신 + 짧은 토스트
- 저장 실패: 원인과 재시도 제공
- 미저장 상태에서 이동: 저장, 저장하지 않고 이동, 취소 제공

### 8.4 Workflow 파싱 오류

정상 컷은 유지하고 오류 컷만 목록화한다. 오류 위치, 원인, 수정 예시를 함께 표시한다.

### 8.5 외부 도구 및 Bridge 실패

현재 컷과 입력값을 보존한다. 실패 상태에서 재시도, 건너뛰기, 도움말 열기를 제공한다.

## 9. 접근성

- 모든 핵심 기능은 키보드로 접근할 수 있어야 한다.
- 포커스 링은 배경과 구분되는 청록색으로 통일한다.
- 탭, 드로어, 바텀시트의 열림 상태를 ARIA 속성으로 전달한다.
- 상태 변화는 색상과 문구를 함께 사용한다.
- 작은 버튼을 줄이고 데스크톱 36px, 모바일 44px 이상의 높이를 확보한다.
- 확대와 모바일 리플로우에서 가로 스크롤이 발생하지 않게 한다.
- 스크린샷만으로 확인할 수 없는 키보드 순서와 보조기기 호환성은 실제 DOM 검증으로 확인한다.

1. SF Studio 전용 디자인 토큰과 공통 Creator Console 셸 추가
2. SF 홈과 Workflow 가져오기 화면 재배치
3. 콘티 제작의 컷 목록, 작업 캔버스, 컨텍스트 패널 재배치
4. 사용방법을 문제 해결 중심으로 재구성
5. 저장, 로딩, 오류, 빈 화면 상태 표현 통일
6. 모바일 하단 메뉴, 컷 드로어, 바텀시트 적용
7. 공개 복사본과 서비스워커 자산 버전 동기화
8. 정적 검사와 실제 브라우저 검증

## 12. 검증 기준

### 기능

- 기존 프로젝트와 이미지가 정상 복원된다.
- 저장, 불러오기, 백업, 복원이 정상 동작한다.
- Workflow MD 가져오기와 오류 이동이 정상 동작한다.
- Midjourney, Grok, Bridge 흐름이 기존과 동일하게 동작한다.
- 홈, 콘티, 가져오기, 사용방법 라우트가 정상 전환된다.

### UI

- 1440px 이상 데스크톱에서 주요 네 영역이 한 화면에 안정적으로 배치된다.
- 390px 모바일에서 가로 스크롤 없이 핵심 작업을 완료할 수 있다.
- 저장, 경고, 오류 상태가 색상 없이도 문구로 구분된다.
- 빈 화면, 로딩, 오류, 완료 상태가 모두 존재한다.

### 자동 검사

- `npm run build`
- `npm run check:workflow-import`
- `npm run check:owner-studio-oauth`
- `npm run check:a11y`
- `npm run check:mobile-css`
- `npm run check:dist`
- 필요 시 전체 `npm run test`

### 운영 경계

- 단일 오너 Google 로그인 구조를 유지한다.
- Cloudflare Access, Firebase, Firestore를 추가하지 않는다.
- Cloudflare Pages 수동 배포는 별도 요청 없이 실행하지 않는다.
- 구현 승인 후 검증된 변경만 일반 커밋과 `origin/main` 푸시 대상으로 삼는다.

## 13. 완료 조건

다음 조건을 모두 충족하면 Creator Console 개편을 완료한 것으로 본다.

1. 네 화면이 동일한 셸과 시각 시스템을 사용한다.
2. 현재 작업과 다음 행동이 각 화면에서 즉시 식별된다.
3. 기존 저장 데이터와 Bridge 흐름에 회귀가 없다.
4. 데스크톱과 모바일에서 주요 작업을 완료할 수 있다.
5. 접근성 및 기존 정적 검사를 통과한다.
6. 배포 후 `https://sunofox.com/mv-studio`에서 오너 로그인과 핵심 화면을 확인한다.
