# SunoFox Site Direction v1

Last locked: 2026-07-05

이 문서는 SunoFox 공개 사이트의 정보 구조와 디자인 방향이 작업 중 흔들리지 않도록 고정하는 기준서입니다. 공개 페이지를 수정할 때는 이 문서를 `AGENTS.md` 다음 우선순위로 확인합니다.

## Fixed Site Type

SunoFox 공개 사이트는 설명이 많은 회사소개형 사이트가 아니라, 스토리 기반 음악과 Story IP를 고급스럽게 전시하는 프리미엄 쇼케이스형 사이트입니다.

핵심 문장:

> SunoFox is a story-driven music and IP studio.

한국어 기준:

> SunoFox는 스토리와 음악을 함께 만드는 IP 스튜디오입니다.

## Audience

| 대상 | 필요한 정보 | 사이트에서의 처리 |
|---|---|---|
| 음악 감상자 | 대표 OST, 앨범, 트랙, 감상 링크 | Music Archive로 빠르게 진입 |
| Story IP 관심자 | 세계관, 작품 분위기, 회차 | Story IP와 에피소드로 진입 |
| 협업/브랜드 확인자 | SunoFox 정체성, 공개 콘텐츠, 운영 기록 | Home, Profile, Updates에서 짧게 확인 |

## Information Priority

공개 페이지는 아래 순서로 정보를 보여줍니다.

1. SunoFox가 무엇인지
2. 대표 콘텐츠 또는 최신 대표 릴리즈
3. Music Archive 진입
4. Story IP 진입
5. 최신 업데이트
6. Profile/About

홈 화면은 특히 1~4번만 강하게 보여주고, 나머지는 보조 정보로 둡니다.

## Page Roles

| Route | 고정 역할 | 주의점 |
|---|---|---|
| `/` | 브랜드 정체성과 대표 콘텐츠를 보여주는 첫 화면 | 많은 카드와 설명을 넣지 않음 |
| `/music/` | OST, 앨범, 트랙을 보여주는 음악 아카이브 | 커버와 앨범 단위를 크게 보여줌 |
| `/music/archive-vol-1/` | 개별 앨범 상세 | 트랙, 분위기, 연결된 Story IP 중심 |
| `/novels/` | Story IP 목록과 세계관 전시장 | 소설 목록 페이지처럼 작게 나열하지 않음 |
| `/novels/episode-*` | 조용하고 읽기 좋은 에피소드 페이지 | 장식보다 가독성과 이전/다음 이동 우선 |
| `/profile/` | SunoFox의 제작 방향과 신뢰 정보 | 이력서처럼 길게 만들지 않음 |
| `/updates/` | 릴리즈, 제작 노트, 공지 | 운영 로그처럼 복잡하게 만들지 않음 |
| `/login/`, `/signup/`, `/account/`, `/admin/`, `/mv-studio/` | 내부/보호 흐름 | 공개 쇼케이스 개편과 분리 |

## Visual Direction

| 항목 | 기준 |
|---|---|
| 전체 인상 | 조용함, 넓은 여백, 큼직함, 프리미엄 쇼케이스 |
| 색상 | 블랙, 오프화이트, 낮은 채도의 포인트 컬러 |
| 타이포그래피 | 큰 제목, 짧은 문장, 명확한 계층 |
| 이미지 | 앨범 커버, IP 포스터, 대표 장면을 크게 사용 |
| 레이아웃 | 섹션 수를 줄이고 한 화면에서 하나의 메시지에 집중 |
| CTA | Music, Story, Listen, Read처럼 짧고 명확하게 |
| 모바일 | 텍스트 겹침 금지, 버튼/카드 높이 안정화, 이미지 우선순위 유지 |

## Do

- 첫 화면에서 SunoFox의 정체성과 대표 콘텐츠가 바로 보여야 합니다.
- 한 섹션에는 하나의 메시지만 둡니다.
- 문장은 짧게 쓰고, 설명은 필요한 곳에만 둡니다.
- 이미지와 타이포그래피가 페이지의 중심이 되게 합니다.
- Music과 Story IP의 진입점은 모든 주요 공개 페이지에서 일관되게 유지합니다.
- 페이지별 수정 전후로 desktop, laptop, mobile 폭에서 텍스트 겹침을 확인합니다.

## Do Not

- 회사소개형 긴 설명으로 첫 화면을 채우지 않습니다.
- 작은 카드 여러 개를 빽빽하게 나열하지 않습니다.
- 대시보드, 운영 콘솔, SaaS 랜딩 페이지처럼 보이게 만들지 않습니다.
- 버튼을 과하게 늘리지 않습니다.
- 텍스트를 이미지 위에 무리하게 겹치지 않습니다.
- 보라색/파란색 그라데이션, 장식용 블롭, 의미 없는 배경 효과를 기본값으로 쓰지 않습니다.
- 페이지마다 서로 다른 스타일의 카드, 버튼, 헤더, 섹션을 만들지 않습니다.

## Page Edit Workflow

1. 한 번에 한 페이지씩 수정합니다.
2. 수정 전 현재 페이지의 문제를 3~5개로 적습니다.
3. 해당 페이지의 고정 역할을 먼저 확인합니다.
4. 필요한 정보만 남기고 나머지는 줄입니다.
5. desktop 1920 또는 1366, mobile 390 기준으로 겹침과 비례를 확인합니다.
6. `npm run build`와 `npm run check`를 통과시킵니다.
7. 검증 통과 후 `AGENTS.md`의 운영 기준에 따라 commit, push, PR, 배포를 진행합니다.

## Acceptance Checklist

| 체크 | 기준 |
|---|---|
| 사이트 정체성 | 첫 화면에서 story-driven music and IP studio로 보이는가 |
| 정보량 | 한 화면에 문장과 버튼이 과하지 않은가 |
| 이미지 | 대표 이미지가 보조가 아니라 중심 역할을 하는가 |
| 여백 | 섹션과 텍스트 사이에 충분한 공간이 있는가 |
| 일관성 | Home, Music, Story IP의 톤이 같은가 |
| 모바일 | 텍스트, 버튼, 이미지가 겹치지 않는가 |
| 이동 | Music / Story / Updates / Profile 진입이 명확한가 |
| 금지선 | 대시보드형, 카드 과다, 긴 설명형으로 흐르지 않았는가 |

## Current Recommended Sequence

1. `/`
2. `/music/`
3. `/novels/`
4. `/music/archive-vol-1/`
5. `/novels/episode-*`
6. `/profile/`
7. `/updates/`

이 순서를 기본값으로 두되, 사용자가 특정 페이지를 지정하면 해당 페이지를 우선합니다.
